# 🕵️‍♂️ 깨비마켓 (Ggaebi Market) QA 및 교차 검증 요청서

안녕하세요! 이 문서는 현재까지 개발된 **깨비마켓 (중고거래 및 경매 하이브리드 앱)**의 프론트엔드 프로토타입 내역을 정리한 QC(데이터 송수신, 코드 리뷰, UI/UX 검증) 요청서입니다. 친구분이 사용 중인 AI(Antigravity)에게 이 문서를 전달하시면, 해당 AI가 전체 맥락을 단숨에 파악하고 고품질의 코드 리뷰와 개선 방향을 제시해 줄 것입니다.

---

## 1. 프로젝트 개요
- **어플명:** 깨비마켓 (Ggaebi Market)
- **컨셉:** 일반 중고거래(고정가/무료나눔)와 '경매' 기능을 결합시킨 혁신적인 모바일 UI 기반의 웹앱. 
- **디자인 모티브:** 직관적이면서도 고급스러운 프리미엄 레이아웃. 글래스모피즘(Glassmorphism), 부드러운 트랜지션 애니메이션, 그리고 트렌디한 네온 그라데이션 포인트.

## 2. 기술 스택
- **프레임워크:** React 18, TypeScript, Vite
- **스타일링:** 순수 CSS (`index.css`를 통한 전역 UI 시스템 제어 및 CSS Variables 활용)
- **아이콘 & 차트:** Lucide-React, Chart.js (`react-chartjs-2`)
- **라우팅 방식:** 별도의 라우터 라이브러리 없이 `App.tsx` 의 최상단 상태(`currentPage`, `history`)를 활용한 SPA 라우팅 및 스크롤 복원 유지 기능 구현.

## 3. 구현된 핵심 페이지 및 기능 명세
현재 모든 코드는 컴포넌트 테스트 및 빠른 반복을 위해 `App.tsx` 파일 내부에 통합 구현되어 있습니다.

1. **`MainPage` (메인 피드)**
   - 스크롤 탭(필터) 및 트렌디한 그리드/리스트 피드 UI.
   - 글래스모피즘이 적용된 고정 상단 헤더 및 하단 네비게이션(Bottoom Nav) 바.
2. **`ProductRegistrationPage` (스마트 상품 등록 마법사)**
   - **Step 1:** 스마트 사진 스튜디오 (AI 누끼, 360도 스캔 모델링 컨셉 UI)
   - **Step 2:** AI 자동 서기 (위치 인식, 상품 정보 자동 채움 컨셉)
   - **Step 3:** 시세 판독 및 판매 방식 선택 (경매, 즉시 구매, 무료 나눔 / `Chart.js`를 이용한 3개월 시세 추이 시 창출)
   - **Step 4:** 정책 체결 통보 화면 (금지 품목, 노쇼 패널티 동의 로직).
3. **`DetailPage` (상품 상세)**
   - 대형 렌더링 사진과 호가 스크롤 패널.
   - 모달 하프 바텀 시트 컨셉의 '입찰하기' 플로우.
4. **마이페이지 및 부가 화면 요소**
   - **`BiddingPage`**: 입찰 참여 내역 현황.
   - **`WishlistPage`**: 찜한 상품 목록 (그리드 배열).
   - **`ChatListPage`**: 거래별 채팅 목록.
   - **`UserPage`**: 사용자 프로필 (매너 온도, 배지, 설정).
   - **`Guides`**: 페널티, 금지 품목, 환경 설정 페이지.

### 🔬 1. 아키텍처 및 리팩토링 검토 (Code Structure)
- **이슈:** 빠른 프로토타이핑을 위해 단일 파일(`App.tsx`)에 대부분의 컴포넌트 코드가 몰려있는 상태입니다.
- **요청 사항:** 기능 단위 / 페이지 단위 / UI 컴포넌트 단위(`src/components`, `src/pages`)로 가장 효율적으로 분할하기 위한 폴더 디렉토리 설계안과 점진적 리팩토링 전략을 제시해 주세요.

### 🎨 2. UI / UX 및 CSS 디자인 퀄리티 체킹
- **이슈:** `max-width: 420px;` 기반의 모바일 퍼스트 뷰로 제작되었습니다.
- **요청 사항:**
  - 모바일 터치 환경에서 버튼 터치 영역(Target Size)이나 여백(Spacing)이 너무 좁거나 부자연스러운 구간이 코드로 보이는지 점검해 주세요.
  - CSS 기반의 글래스모피즘 효과와 애니메이션(`fade-slide-up` 등) 최적화 상태를 리뷰해 주세요. 접근성(A11y)과 컬러 대비가 충분한지도 확인 부탁드립니다.

### ⚙️ 3. 상태 관리 및 성능 분석
- **이슈:** `useState`만으로 모든 라우팅과 `scrollPositions`(스크롤 유지)을 제어하고 있습니다.
- **요청 사항:** 스크롤 복원 등 현재 구현된 야생(Vanilla) 라우팅 방식에서 야기될 수 있는 메모리 누수나 렌더 타이밍(리렌더링 병목) 현상이 있는지 점검해 주세요. 향후 `react-router-dom`과 클라이언트 상태 관리자(Zustand 등) 도입 시 구조를 어떻게 이관하면 좋을지 논의해 주세요.
